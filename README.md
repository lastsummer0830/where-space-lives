## 담당 역할 — 조장 · 서브 컨셉 페이지 · 상품 상세

팀 4명 중 **조장**을 맡아 협업 흐름을 설계하고, 화면에서는 **서브 컨셉 페이지와 상품 상세**를 전담했습니다. (메인 화면 · 헤더 · 로그인 · 상품 목록은 팀원이 담당했습니다.)

**협업 (조장으로서)**
- `feature` 브랜치 + Pull Request 워크플로우를 도입해, 팀원 각자가 독립 브랜치에서 작업하고 PR로 합치도록 운영했습니다.
- feature 브랜치 기반 협업으로 팀 전체 **PR 65건**을 리뷰·병합하며 `main`을 관리했습니다. 여러 명이 같은 정적 사이트를 만지면서도 충돌을 통제했습니다. (git 이력에 그대로 남아 있습니다.)

**화면 (직접 구현)**
- **컨셉 페이지 4종** — 우드(`햇살이 머무는 우드 컬렉션`) · 다크(`Dark comfort, clean design`) · 코지(`Cozy & Warm`) · 네이처(`자연을 닮은 휴식처`). 테마별로 상품을 묶어 보여주는 큐레이션 화면입니다.
- **상품 상세 페이지** — 가격 · **색상 선택** · 상세 이미지(매트리스 상세 6컷 등)로 구성한 단일 상품 화면.
- **스크롤 인터랙션** — 컨셉 페이지에서 스크롤에 따라 요소가 나타나는 애니메이션(`slide-left`, `fade-up`)을 직접 붙였습니다.
- **이미지 에셋** — 상품 · 컨셉 이미지 정리 및 `.avif` 최적화 적용.

## 트러블슈팅 — 스크롤 애니메이션이 화면마다 다르게 동작한 문제

- **상황**: 네이처 컨셉 페이지는 스크롤에 따라 요소가 나타나는 애니메이션(`animate-item` → `.show` 클래스 부여)으로 만들었는데, **내 화면에서는 특정 요소가 제때 안 나타나거나 어긋나게 보였습니다.**
- **파고든 지점**: 이 페이지는 `window`가 아니라 별도의 `.scroll-container`(세로 `scroll-snap`) 안에서 스크롤이 일어나는 구조라, 요소가 "화면에 들어왔는지" 판정하는 기준이 뷰포트 크기 · 스크롤 위치에 따라 달라졌습니다.
- **해결 과정**: 내 화면만으로는 재현 조건이 헷갈려서, **같은 코드가 팀원 화면에서는 어떻게 보이는지 직접 대조**했습니다. 화면 크기에 따라 증상이 갈리는 걸 확인하고, 애니메이션 트리거 기준과 대상 요소를 맞춰 어느 화면에서도 같게 나타나도록 수정했습니다.
- **배운 점**: 스크롤/뷰포트에 의존하는 UI는 **내 환경 하나로 판단하면 안 된다**는 것. 이후로는 화면 크기를 바꿔 보거나 다른 사람 환경에서 확인하는 습관이 생겼습니다.

## 실행 방법

별도 빌드 과정 없이 동작하는 **정적 사이트**입니다.

```bash
git clone https://github.com/lastsummer0830/where-space-lives.git
cd where-space-lives
```

`index.html`을 브라우저로 바로 열어도 되지만, 일부 화면이 **ES Module**로 스크립트를 나눠 불러오므로 로컬 정적 서버로 여는 것을 권장합니다.

```bash
npx serve .          # 또는 VS Code의 Live Server 확장
```

→ 브라우저에서 `index.html` (메인) · `sub_screen/html/SubProject.html` (우드 컨셉) 등으로 진입
